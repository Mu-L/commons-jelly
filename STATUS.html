1. INTRODUCTION

The Jelly package implements a simple XML tag based scripting engine. It borrows many ideas from both JSP custom tags, Velocity and the scripting engine inside XDoclet. Jelly can be used from the command line, inside Ant or inside a Servlet. It allows the engine to be extended using custom tags in a similar way to JSP. Though Jelly is really simple and has no dependencies on JSP.

As an experiment some of the JSTL tags are implemented.

2. DEPENDENCIES

The Jelly package is dependent upon the following external components for development and use:

  • Java Development Kit (Version 1.2 or later)
  • Jakarta Commons Collections (Version 1.1 or later)
  • Jakarta Commons BeanUtils (Version 1.1 or later)
  • Jakarta Commons Logger (Version 1.0 or later)
  • JUnit Testing Framework (Version 3.7 or later) - for unit tests only, not required for deployment

The XML tags currently depend on

  • dom4j (Version 1.1 or later)
